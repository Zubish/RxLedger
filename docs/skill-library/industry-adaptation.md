# Industry Adaptation

Use this skill when applying RxLedger lessons to a different industry.

## Adaptation Method

1. Name the industry and primary users.
2. Map physical objects and records.
3. Map operating sites and teams.
4. Map time-sensitive risks.
5. Map customer continuity needs.
6. Map compliance or audit requirements.
7. Map subscription boundaries.

## Industry Mapping Examples

| Industry     | Inventory/Record                      | Time Risk                            | Continuity Record         | Site Context           |
| ------------ | ------------------------------------- | ------------------------------------ | ------------------------- | ---------------------- |
| Restaurant   | Ingredients, prepared orders          | Freshness, table wait time           | Customer/order history    | Branch, kitchen, table |
| Clinic       | Supplies, appointments, patient files | Appointment delay, medication expiry | Patient chart             | Clinic, department     |
| Auto repair  | Parts, job cards                      | Warranty, service SLA                | Vehicle service history   | Workshop, bay          |
| School       | Student records, fees, attendance     | Term deadlines                       | Student profile           | Campus, class          |
| Beauty salon | Products, bookings, staff schedules   | Appointment time, product stock      | Client treatment history  | Outlet, stylist        |
| Logistics    | Parcels, routes, fleet                | Delivery SLA                         | Customer shipment history | Hub, driver route      |
| Construction | Materials, equipment, tasks           | Permit dates, rental return          | Project history           | Site, contractor       |

## Domain Translation Rules

- Do not copy pharmacy words into another domain.
- Preserve the pattern behind the word.
- Replace "branch" with the domain's real operating unit.
- Replace "patient" with the domain's continuity entity.
- Replace "batch/expiry" with the domain's risk-bearing inventory unit.
- Replace "dispense/sale" with the domain's transaction moment.

## Feature Prioritization

Start with:

- Core records.
- Search/retrieval.
- Transaction workflow.
- History.
- Reports.
- Roles.

Then add:

- Reminders.
- Follow-up communication.
- Cross-site coordination.
- Advanced analytics.
- AI support.

## Pricing Boundary Guidance

- Low plan: one site, core workflow, basic reports.
- Growth plan: multiple sites, automation, staff controls, communication.
- Enterprise plan: unlimited scale, integrations, custom reporting, SLA, onboarding.

The boundary should protect high-cost features while keeping the entry plan useful.
