# UI and UX Design

Use this skill when changing visual design, layout, landing pages, dashboards, or dense app screens.

## Product Surface Type

Choose the design posture based on the surface:

- Landing page: trust, brand, proof, clarity, conversion.
- Authenticated workspace: calm, dense, operational, fast to scan.
- POS or task tool: focused, stable, low distraction.
- Reports: comparable, exportable, filterable.
- Settings/admin: restrained, precise, hard to misuse.

## Layout Principles

- Give each page one primary job.
- Keep repeated operational cards compact.
- Avoid cards inside cards unless the inner card is a repeated record.
- Do not use decorative sections inside authenticated workspaces.
- Align CTA spacing by content rhythm, not by forcing buttons to the bottom when it creates empty gaps.
- Make expandable content feel natural: a text button can be better than a boxed button for secondary reveal actions.

## Visual Asset Principles

- Use real or generated bitmap images when they make a product, place, person, or workflow more concrete.
- A background image must support the content, not compete with it.
- Use overlays and text shadows when placing text over images.
- Verify text readability on desktop and mobile.
- Avoid generic stock-like imagery when the user needs trust or specificity.
- Prefer local optimized assets over hotlinking for production stability.

## Brand and Color

- Use color as a system, not decoration.
- Avoid one-hue pages.
- Green can signal operational trust, but pair it with restrained neutrals and occasional accent colors.
- Keep alert colors meaningful: danger, warning, success, information.
- Do not overuse gradients or ornamental backgrounds.

## Typography and Density

- Use large type only for true hero or major landing sections.
- Use compact headings in cards, dashboards, sidebars, and tools.
- Avoid viewport-scaled text.
- Ensure text fits on mobile.
- Use plain language for action buttons.

## Landing Page Pattern

- First viewport should communicate the product immediately.
- Pricing cards should be scannable before expanded.
- Testimonials become stronger with a relevant environment image and readable overlay.
- FAQs should remove buying friction, not repeat marketing claims.

## Accessibility

- Icon buttons need labels or titles.
- Interactive toggles need state where relevant, such as `aria-expanded`.
- Text over images needs strong contrast.
- Do not rely only on color to convey status.
